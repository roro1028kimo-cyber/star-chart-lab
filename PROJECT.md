# PROJECT.md

## 專案名稱
- 暫名：`star-chart-lab`

## 目前聚焦範圍
- 主題：星座、星盤、西洋占星
- 先不做：
  - 塔羅
  - 紫微斗數
  - 八字
  - 過多的每日運勢內容站

## 功能優先順序
1. 個人本命盤計算
2. 十二宮位置與主要行星落點分析
3. 運勢分析
4. AI 個人化建議

## 核心使用流程
1. 使用者進入首頁
2. 輸入出生日期、出生時間、出生地
3. 系統根據星歷表基準計算本命盤資料
4. 顯示十二宮、主要行星落點與基礎解讀
5. 後續階段再延伸運勢分析與 AI 建議

## MVP 定義
- 輸入欄位
  - 出生日期
  - 出生時間
  - 出生地
- 結果內容
  - 太陽星座
  - 月亮星座
  - 上升星座
  - 十二宮位置
  - 主要行星落點
  - 基本解讀區塊
- 第一版先不做
  - 每日 / 每週 / 每月運勢
  - AI 即時問答或個人化建議
  - 分享圖卡
  - 登入功能
  - 過度複雜的社群或內容站架構

## 第一版目標
- 先完成可驗證的本命盤計算閉環
- 先把「客觀位置計算」與「主觀解讀文案」分開
- 讓結果頁先能穩定呈現基本本命盤摘要

## 資料來源方向
- 星歷表 / 本命盤計算基準：
  - 實作採用：`circular-natal-horoscope-js`
  - 查核參考：JPL Horizons、Astronomical Almanac 類型資料
  - 若未來要改更高精度路線，可再回到 Swiss Ephemeris 直連方案
- 地理與時區：
  - 地點搜尋：Nominatim
  - 地點 / 時區解算：以經緯度直接做本地換算，不要求使用者註冊外部帳號
- 解讀內容：
  - 第一版以規則庫與硬編碼文案驗證
- AI 建議：
  - 第二階段再接 API
  - 優先找推理穩定、結構化輸出好接的模型

## 產品原則
- 先做正確的資料流，再做華麗包裝
- 先驗證輸入到結果的閉環，再擴張更多占星功能
- 將「客觀天文位置」與「主觀占星解讀」分開處理
